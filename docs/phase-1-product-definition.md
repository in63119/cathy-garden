# Phase 1. 제품 정의

## 목적

Cathy Garden을 아내 1인을 위한 개인용 사진/영상 보관함으로 정의한다.  
이 단계의 목표는 무엇을 만들지와 무엇을 아직 만들지 않을지를 명확히 고정하는 것이다.

## 제품 한 줄 정의

`Cathy Garden`은 휴대폰에 있는 사진과 동영상을 업로드하고, 정리하고, 다시 꺼내볼 수 있는 개인용 private archive다.

## 핵심 사용자

- 주 사용자: 아내 1명
- 운영/관리: 남편 1명

현재 단계에서는 불특정 다수를 위한 서비스가 아니다.  
가입형 서비스, 공개형 서비스, 소셜 서비스로 정의하지 않는다.

## 제품 목표

- 사진과 동영상을 안전하게 업로드할 수 있어야 한다.
- 업로드한 파일을 모바일에서 쉽게 다시 볼 수 있어야 한다.
- 개인 공간처럼 느껴지는 따뜻한 UI를 제공해야 한다.
- 복잡한 기능보다 사용성과 단순함을 우선해야 한다.
- 이후 기능을 점진적으로 추가할 수 있는 구조여야 한다.

## 비목표

현재 단계에서 아래 항목은 목표에 포함하지 않는다.

- 공개 공유 중심 서비스
- 다중 사용자 권한 체계
- 소셜 피드
- 댓글/좋아요
- NFT, 민팅, 마켓 기능
- 무거운 편집 기능
- 복잡한 관리자 백오피스

## MVP 범위

초기 출시 범위는 아래로 제한한다.

- 로그인
- 사진 업로드
- 동영상 업로드
- 업로드 목록 보기
- 상세 보기
- 삭제
- 날짜순 정렬

## 사용자 시나리오

### 시나리오 1. 업로드

1. 사용자가 로그인한다.
2. 휴대폰에서 사진 또는 동영상을 선택한다.
3. 파일을 업로드한다.
4. 업로드가 완료되면 보관함 목록에 반영된다.

### 시나리오 2. 보관함 확인

1. 사용자가 보관함에 들어간다.
2. 업로드된 사진과 동영상을 목록에서 본다.
3. 특정 항목을 눌러 상세 화면으로 이동한다.
4. 사진은 크게 보고, 동영상은 재생한다.

### 시나리오 3. 정리

1. 사용자가 불필요한 사진이나 영상을 선택한다.
2. 삭제를 실행한다.
3. 목록에서 즉시 반영된다.

## 기능 요구사항

### 반드시 필요한 기능

- 인증된 사용자만 접근 가능해야 한다.
- 사진 업로드가 가능해야 한다.
- 동영상 업로드가 가능해야 한다.
- 업로드 상태를 사용자에게 보여줘야 한다.
- 업로드된 항목을 목록으로 보여줘야 한다.
- 사진/영상 상세 보기가 가능해야 한다.
- 항목 삭제가 가능해야 한다.

### 있으면 좋은 기능

- 촬영일 기준 정렬
- 썸네일 표시
- 업로드 실패 재시도
- 빈 상태 안내 화면

### 이후 단계 기능

- 앨범
- 태그
- 검색
- 즐겨찾기
- 공유 링크
- 자동 썸네일 생성
- 중복 업로드 감지

## UX 방향

### 핵심 톤

- 전원주택
- 정원
- 개인적인 공간
- 따뜻함
- 차분함

### 피해야 할 방향

- 일반적인 SaaS 대시보드 느낌
- 차갑고 기계적인 업로드 툴 분위기
- NFT/크립토 제품처럼 보이는 UI
- 복잡한 정보 밀도

### 화면 원칙

- 모바일 우선
- 큰 터치 영역
- 단순한 동선
- 사진 중심 레이아웃
- 따뜻한 자연색 계열 사용

## 콘텐츠 원칙

- 제품 문구는 기술 제품처럼 쓰지 않는다.
- 사용자가 느끼기에 `클라우드 스토리지`보다 `우리만의 보관함`에 가깝게 표현한다.
- 업로드, 보관, 다시 보기라는 핵심 행동을 직관적으로 드러낸다.

예시 표현:

- 우리의 정원 보관함
- 사진과 영상을 차곡차곡
- 집처럼 편안한 개인 아카이브

## 접근 정책

- 기본적으로 비공개 서비스로 운영한다.
- 허용된 사용자만 접근 가능해야 한다.
- 개인 정보와 미디어 파일 접근은 보수적으로 다룬다.
