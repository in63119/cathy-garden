# 공모전 달력 및 제출물 아카이브 태스크

## 목적

메인 페이지에서 공모전 일정을 바로 확인하고, 캡쳐한 공모전 정보를 달력과 연결해 관리할 수 있도록 한다.

이 기능의 목표는 공모전 정보를 놓치지 않고 모아두며, 아이디어 메모와 제출물을 함께 보관하는 개인용 공모전 관리 흐름을 만드는 것이다.

## 핵심 요구사항

- 달력을 메인 페이지 메뉴 바로 밑에 배치한다.
- 공모전 달력 항목을 클릭하면 해당 공모전 상세 정보가 보여야 한다.
- 공모전 상세에는 캡쳐한 공모전 이미지와 핵심 정보를 함께 표시한다.
- 공모전 목록은 S3 JSON에서 불러오고, 등록/수정/삭제할 수 있어야 한다.
- 공모전별 아이디어 메모장을 제공한다.
- 제출 완료 또는 준비 중인 제출물을 아카이브할 수 있어야 한다.

## 화면 구성

### 메인 페이지

- 상단 메뉴 바로 아래에 달력 영역을 배치한다.
- 로그인하지 않은 사용자는 공모전 달력을 볼 수 없다.
- 달력에는 공모전 마감일 또는 주요 일정이 표시되어야 한다.
- 공모전이 있는 날짜는 일반 날짜와 구분되도록 표시한다.
- 일정이 없는 날짜를 누르면 해당 날짜를 제출 마감일로 하는 공모전 등록 폼이 열린다.
- 모바일에서도 메뉴 아래에서 바로 확인 가능해야 한다.

### 공모전 상세

달력에서 공모전 일정을 클릭하면 아래 정보를 확인할 수 있어야 한다.

- 공모전 이름
- 마감일 또는 제출 기한
- 상금 규모
- 캡쳐한 공모전 이미지
- 아이디어 메모장
- 제출물 아카이브 연결

## 데이터 항목

공모전 항목은 최소한 아래 데이터를 가진다.

- `title`: 공모전 이름
- `deadline`: 제출 마감일
- `prize`: 상금 정보
- `captureImage`: 캡쳐 이미지 경로 또는 업로드 파일 정보
- `ideaMemo`: 아이디어 메모 내용
- `submissions`: 제출물 목록
- `createdAt`: 등록일
- `updatedAt`: 수정일

공모전 목록은 `contests/index.json`에 저장한다.
공모전별 부가 데이터는 아래 경로를 사용한다.

- `contests/{contestId}/idea-memo.json`: 아이디어 메모
- `contests/{contestId}/submissions.json`: 제출물 아카이브
- `contests/{contestId}/capture.png`: 캡쳐 이미지 예시 경로

## 사용자 시나리오

### 시나리오 1. 달력에서 공모전 확인

1. 사용자가 메인 페이지에 들어온다.
2. 메뉴 바로 아래의 달력을 확인한다.
3. 공모전 일정이 표시된 날짜를 누른다.
4. 해당 공모전의 상세 정보를 확인한다.

### 시나리오 2. 캡쳐한 공모전 정보 보기

1. 사용자가 공모전 달력에서 항목을 선택한다.
2. 공모전 상세 화면이 열린다.
3. 캡쳐 이미지, 공모전 이름, 마감일, 상금 정보를 확인한다.

### 시나리오 3. 아이디어 메모 작성

1. 사용자가 공모전 상세 화면에 들어간다.
2. 아이디어 메모장에 아이디어를 작성하거나 수정한다.
3. 저장 후 다시 들어와도 메모가 유지된다.

### 시나리오 4. 제출물 아카이브

1. 사용자가 공모전 상세 화면에서 제출물 아카이브를 연다.
2. 제출 파일 또는 제출 관련 이미지를 등록한다.
3. 등록한 제출물을 공모전별로 다시 확인한다.

### 시나리오 5. 공모전 관리

1. 사용자가 달력에서 공모전을 등록할 날짜를 누른다.
2. 공모전 이름, 제출 마감일, 상금, 캡쳐 이미지 object key를 입력한다.
3. 등록한 공모전이 달력에 표시된다.
4. 필요한 경우 기존 공모전을 수정하거나 삭제한다.

## 제출물 아카이브 범위

제출물 아카이브는 공모전별 결과물을 보관하는 공간으로 정의한다.

초기 범위는 아래로 제한한다.

- 공모전별 제출물 목록 보기
- 제출물 파일 또는 이미지 등록
- 제출물 이름 표시
- 제출일 또는 등록일 표시
- 제출물 상세 보기

이후 확장 기능은 아래로 둔다.

- 제출 상태 관리
- 수상 여부 기록
- 제출 링크 저장
- 버전별 제출물 관리
- 제출물 검색

## UX 원칙

- 달력은 메인 페이지의 보조 기능이 아니라 주요 진입점으로 보여야 한다.
- 공모전 정보는 한눈에 읽히도록 이름, 마감일, 상금 정보를 우선 노출한다.
- 캡쳐 이미지는 원본 내용을 확인할 수 있을 만큼 크게 표시한다.
- 아이디어 메모장은 별도 복잡한 에디터보다 빠르게 쓰고 저장하는 흐름을 우선한다.
- 제출물 아카이브는 사진/영상 보관함의 기존 정체성과 충돌하지 않도록 차분하고 개인적인 보관 경험으로 만든다.

## 비목표

현재 단계에서는 아래 기능을 포함하지 않는다.

- 공개 공모전 검색 서비스
- 외부 공모전 사이트 자동 크롤링
- 다중 사용자 협업
- 심사 결과 자동 추적
- 공개 포트폴리오 페이지
- 복잡한 프로젝트 관리 보드

## 구현 고려사항

- 개인 데이터와 제출물 파일은 비공개 접근 정책을 유지해야 한다.
- 공모전 캡쳐 이미지와 제출물은 기존 업로드/스토리지 구조를 재사용할 수 있는지 먼저 확인한다.
- 달력 UI는 모바일 터치 영역을 충분히 확보해야 한다.
- 메모 저장은 자동 저장 또는 명시적 저장 중 하나를 선택해 사용자 경험을 일관되게 유지한다.
- 테스트는 `tests/agent/` 아래에 기능 범위에 맞춰 추가하거나 갱신한다.

## 완료 기준

- 메인 페이지 메뉴 바로 아래에 달력이 배치된다.
- 로그인하지 않은 사용자에게는 공모전 달력이 표시되지 않는다.
- 달력에서 공모전 일정을 클릭하면 상세 정보가 열린다.
- 달력에서 일정이 없는 날짜를 클릭하면 공모전 등록 폼이 열린다.
- 상세 정보에서 공모전 이름, 마감일, 상금, 캡쳐 이미지를 확인할 수 있다.
- 공모전별 아이디어 메모를 작성하고 저장할 수 있다.
- 공모전별 제출물 아카이브를 확인할 수 있다.
- 공모전 목록을 S3 JSON에서 불러올 수 있다.
- 공모전을 등록, 수정, 삭제할 수 있다.
- 관련 테스트가 추가 또는 갱신되고 실행 결과가 확인된다.
