# Phase 5. 디자인 시스템 및 브랜딩

## 목적

이 문서는 Cathy Garden의 시각적 방향과 UI 원칙을 고정한다.

이 프로젝트는 일반적인 SaaS 대시보드처럼 보이면 안 된다.  
아내 1인을 위한 개인 사진/영상 보관함이라는 성격과, `public/logo512.png`에 담긴 전원주택과 텃밭의 분위기를 화면 전반에 반영해야 한다.

## 브랜딩 한 줄 정의

`Cathy Garden`은 차갑고 기능적인 클라우드 저장소가 아니라, 집처럼 편안한 개인용 사진 보관함이다.

핵심 인상은 다음과 같다.

- 따뜻함
- 조용함
- 정원과 집의 생활감
- 개인적인 기록 공간

## 비주얼 방향

### 피해야 하는 방향

- 일반적인 B2B 대시보드 톤
- 강한 파란색 중심의 클라우드 서비스 느낌
- 차갑고 유리 같은 인터페이스
- 보라색 포인트 중심의 흔한 AI/SaaS 스타일
- 과하게 기술적인 용어가 드러나는 화면

### 지향하는 방향

- 햇빛이 드는 전원주택과 정원 분위기
- 부드러운 베이지, 세이지 그린, 우드 브라운 계열
- 사진이 잘 드러나는 여백 중심 레이아웃
- “저장소”보다 “우리 집 앨범”에 가까운 인상
- 모바일에서 손에 잡히는 큰 버튼과 단순한 흐름

## 브랜드 키워드

- Country home
- Garden archive
- Warm privacy
- Calm mobile experience
- Personal memory shelf

## 기본 톤 앤 매너

### 언어 톤

- 기능 설명은 짧고 부드럽게 쓴다.
- “media asset”, “storage orchestration” 같은 기술 용어는 사용자 화면에서 최소화한다.
- “보관함”, “사진”, “영상”, “기록”, “정원”, “집” 같은 단어를 우선한다.

### 화면 인상

- 첫 화면은 기능보다 분위기를 먼저 전달한다.
- 목록 화면도 파일 관리자처럼 보이지 않게 한다.
- 업로드 화면은 작업 도구라기보다 안내된 개인 공간처럼 느껴져야 한다.

## 컬러 시스템

현재 구현과 앞으로의 확장을 고려한 기본 컬러 토큰은 아래를 기준으로 한다.

```css
:root {
  --background: #eef3ea;
  --surface: rgba(255, 252, 246, 0.82);
  --surface-strong: #f8f3ea;
  --foreground: #243126;
  --muted: #5a6d59;
  --accent: #6f8b62;
  --accent-strong: #4c6942;
  --border: rgba(76, 105, 66, 0.18);
  --shadow: 0 18px 40px rgba(55, 74, 48, 0.12);
}
```

색상 역할은 다음과 같다.

- `--background`
  - 정원 공기감이 느껴지는 전체 배경
- `--surface`
  - 카드와 패널 배경
- `--foreground`
  - 기본 텍스트
- `--muted`
  - 보조 설명 텍스트
- `--accent`
  - 포인트 컬러
- `--accent-strong`
  - 주요 CTA 버튼과 강조 텍스트
- `--border`
  - 카드 윤곽과 구분선

## 타이포 방향

현재는 `Georgia, Times New Roman, serif`를 기준으로 쓰고 있다.

이 방향은 프로젝트 성격과 잘 맞는다.

- 지나치게 현대적이거나 스타트업스러운 느낌을 줄인다.
- 집과 기록, 앨범 같은 인상을 강화한다.
- 큰 제목에서 감성적인 무드를 만들기 쉽다.

원칙은 다음과 같다.

- 제목은 존재감 있게 크게 쓴다.
- 본문은 과도하게 작게 줄이지 않는다.
- 모바일에서도 읽기 쉬운 줄간격을 유지한다.
- 화면 전체에 폰트 종류를 많이 섞지 않는다.

## 이미지 사용 원칙

`public/logo512.png`는 단순 로고 이미지가 아니라 현재 브랜드 무드를 정의하는 핵심 시각 자산으로 본다.

활용 원칙:

- 홈 화면 hero에서 크게 사용한다.
- 사진/영상 UI보다 먼저 브랜드 분위기를 잡는 역할을 맡긴다.
- 배경 이미지를 억지로 반복 사용하지 않는다.
- 카드 안 대표 이미지나 소개 섹션에서 제한적으로 쓴다.

주의:

- 작은 아이콘처럼 축소해 의미를 잃게 만들지 않는다.
- 업로드 목록에서 반복 장식으로 남발하지 않는다.

## 레이아웃 원칙

### 전체 레이아웃

- 세로 스크롤 중심
- 과도한 사이드바 구조 지양
- 카드 중심의 단순한 정보 묶음
- 여백을 충분히 사용

### 모바일 우선 원칙

- 주요 버튼은 손가락으로 누르기 쉽게 충분히 크게 만든다.
- 필터/정렬은 복잡한 테이블 도구처럼 만들지 않는다.
- 목록은 카드 그리드 또는 단순 리스트로 유지한다.
- 화면당 핵심 행동은 1~2개를 넘기지 않는다.

## 컴포넌트 스타일 원칙

### 카드

- 둥근 모서리
- 연한 배경
- 약한 경계선
- 과하지 않은 그림자

카드는 “패널”보다 “작은 종이 카드”에 가까운 인상을 목표로 한다.

### 버튼

- 메인 액션은 진한 초록 계열
- 보조 액션은 밝은 표면색과 경계선 사용
- 버튼 모양은 너무 각지지 않게 유지
- 모바일 탭 영역을 충분히 확보

### 입력 요소

- 업로드와 로그인 입력창도 차갑게 보이지 않게 한다.
- 선만 있는 입력보다 연한 배경과 부드러운 외곽선을 우선한다.
- 에러 상태는 자극적인 빨강보다 읽기 쉬운 대비 중심으로 처리한다.

## 페이지별 디자인 목표

### 홈

- 브랜드 분위기 전달
- 이미지와 큰 카피 중심
- 기술 설명보다 정체성 우선

### 로그인

- “접근 제한된 개인 공간”이라는 의미 전달
- 불필요한 공포감 없이 차분한 보호 느낌 유지

### 업로드

- 작업 흐름이 분명해야 함
- 파일 선택, 업로드 진행, 성공 상태가 한눈에 보여야 함
- 작업 도구이지만 여전히 브랜드 톤 유지

### 보관함

- 파일 관리자보다 앨범형 인상
- 사진이 중심이 되고 메타데이터는 보조 정보로 남음
- 영상은 무거운 미리보기보다 대표 카드 전략 우선

### 상세 보기

- 사진/영상 자체를 가장 크게 보여준다.
- 삭제 같은 파괴적 액션은 분명하지만 과장하지 않는다.
- 메타데이터는 보조 카드나 정리된 섹션에 둔다.

## 모션 원칙

- 필요한 곳에만 쓴다.
- 업로드 성공, 카드 hover, 화면 진입 정도에 제한한다.
- 과한 bounce, spring, flashy transition은 지양한다.

목표는 “조용한 반응성”이다.

## 접근성 원칙

- 텍스트 대비를 충분히 확보한다.
- 버튼과 링크의 클릭 영역을 넉넉히 둔다.
- 색만으로 상태를 구분하지 않는다.
- 업로드/삭제/로그인 상태 메시지는 문장으로 같이 전달한다.

## 현재 구현과의 연결

현재 전역 스타일은 이미 이 방향의 초안을 반영하고 있다.

- 배경: 연한 그린/베이지 계열 그라데이션
- 카드: 반투명 표면 + 둥근 모서리
- CTA: 짙은 그린 계열
- hero: 전원주택 이미지 중심

Phase 5의 목적은 이 초안을 취향 수준이 아니라, 반복 가능한 규칙으로 고정하는 것이다.

## 완료 기준

아래 조건을 만족하면 Phase 5를 완료로 본다.

- 컬러 토큰이 문서와 코드에서 일치한다.
- 홈, 로그인, 업로드, 보관함, 상세 보기에 일관된 톤이 반영된다.
- 버튼, 카드, 입력 UI에 공통 스타일 규칙이 적용된다.
- 모바일 화면에서도 분위기와 사용성이 함께 유지된다.
- 이후 기능을 추가해도 같은 브랜드 방향을 유지할 수 있다.

## 다음 단계 연결

이 문서를 기준으로 다음 작업에서 다루어야 할 항목은 아래와 같다.
